179 Versions

Kong Interactive isn't a single piece — it's a living system that has evolved through 179 distinct iterations. Each version refines the relationship between visitor input and generative output.

This iterative approach is central to how I work. Rather than designing a perfect system on paper, I build, test with real visitors, observe what works, and evolve. The 179th version knows things the 1st version couldn't.

Multi-Sensor Input

Kong uses multiple input sources simultaneously: depth cameras for body tracking, environmental sensors for ambient conditions, and gestural analysis for movement quality. These streams feed a modular TouchDesigner pipeline that generates unique visuals for every interaction.

Depth Cameras

Body tracking, blob detection, depth-to-particles conversion.

Gesture Analysis

Movement quality detection: speed, smoothness, expansion.

Modular TD Pipeline

Reusable component architecture. Each module can be swapped independently.

Generative Output

Particle systems, feedback loops, instancing. Never the same twice.

Component Library

Kong has been the primary testing ground for developing reusable interactive components. Many of the modules born in Kong — skeleton tracking, depth-to-particles, feedback systems — have been extracted into a growing library used across other projects.

This approach means each new project doesn't start from scratch. Proven components are combined and extended, dramatically reducing development time while maintaining quality.
